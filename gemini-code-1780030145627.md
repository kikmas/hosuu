# 【引き継ぎ仕様書】歩数分割スケール可変シミュレーター（ポケセン風UI）

本プロジェクトは、特定の歩数（X軸・Y軸）を設定し、それを任意に等分した際の「1マスあたりの歩数」や「画面比率」をリアルタイムに視覚化・計算するWebツールの完成版ソースコードです。

新規開発の続行や機能追加の際は、以下の仕様・デザインシステム・実装構造を厳守してください。

---

## 1. ツール概要・目的
ゲームなどのマップ検証において、「横X歩、縦Y歩」のエリアをそれぞれ任意の数で等分（グリッド化）した際、1マスマップが何歩に相当するのかを直感的にシミュレーションするためのツールです。

## 2. デザインシステム（ポケセン風・レトロポップ）
懐かしいゲームのUIをリスペクトしたポップでクッキリしたデザインを採用しています。
* **メインカラー**:
    * `--poke-red`: `#E3350D`（ヘッダー、主要ボタン、結果数値）
    * `--poke-blue`: `#316CB6`（入力枠、ラベル、グリッド線）
    * `--poke-yellow`: `#FFCB05`（背景、結果セクション背景）
    * `--poke-dark`: `#2A2A2A`（全体の外枠線、フォント色）
* **フォント**: `'M PLUS Rounded 1c', sans-serif`（丸ゴシック系、Google Fontsより読み込み）
* **共通装飾**: 外枠はすべて `3px`〜`4px` の太い実線（`#2A2A2A`）で縁取り、ポップな立体感（シャドウ）を付与。背景には45度のストライプ模様を配置。

---

## 3. UI・操作仕様（重要）

### ① そう歩数（X/Y）の入力：大型スロットドラム（リール）式
* スマートフォンでの操作性を最優先し、テキスト入力を廃止。
* 上下の**マウスホイール回転**、または指・マウスでの**上下スワイプ（ドラッグ）**によって数値を1ずつ増減させる独自UI。
* **ドラムサイズ**: 可認性を高めるため大型化（縦52px、横85px、フォント1.4rem）。上下に立体感を出すグラデーションシャドウを配置。

### ② とう分数（X/Y）の入力：左右矢印ボタン式
* ドラム操作との誤操作・競合を防ぐため、こちらは**「◀」「▶」のクリック（タップ）のみ**で1ずつ増減する仕様。

### ③ 計算結果表示エリア
* X軸・Y軸それぞれの「1マスあたりの歩数（総歩数 ÷ 等分数）」を小数点第1位（四捨五入）まで計算し、リアルタイム表示。

### ④ 視覚化キャンバス（HTML5 Canvas）
* 設定された「そう歩数 X : Y」の比率を保ったまま、枠内に収まる最大サイズで白い四角形（マップ）を動的描画。
* マップ内は指定された「とう分数」でブルーの格子線（グリッド）を描画。
* 各マスの中央には、何番目のマス目かがわかるインデックス（例: `1,1` `2,3`）をクッキリとした白縁取り文字で描画（ただし、マスが潰れるのを防ぐため、**等分数が「縦横ともに15以下」の場合のみ自動描画**される仕様）。
* キャンバスの左側と下部には、現在の総歩数（0歩 〜 最大歩数）の目盛りを動的に描画。

## 4. モバイル（スマートフォン）最適化仕様
ドラムコントロールをスマホでスワイプする際、ブラウザ全体の画面スクロール（ガタつき）と競合するバグを完全に解決するため、以下の防衛策が施されています。機能追加時もこの構造を崩さないでください。

1.  **CSSによるスクロールブロック**:
    `.slot-reel` 要素に対して `touch-action: none;` を指定し、ブラウザのデフォルトスクロールを完全にシャットアウト。
2.  **JSによるイベント割り込み**:
    `touchmove` イベント内で `e.preventDefault()` を強制実行（`{ passive: false }` でリスナーを登録）。これにより指の動きが100%ドラムの回転処理へと変換されます。

---

## 5. 技術スタック・制限値
* **言語**: 純粋な HTML5 / CSS3 / Vanilla JavaScript（外部フレームワーク依存なし）
* **数値制限（デフォルト）**:
    * そう歩数 (X / Y) : `min="1"`, `max="150"`（初期値: X=25, Y=22）
    * とう分数 (X / Y) : `min="1"`, `max="20"`（初期値: X=4, Y=3）
* **キャンバス解像度**: ベースサイズは `540 × 540`（CSSでレスポンシブ縮小）。文字の潰れやバグ（`undefined`によるキャンバス消失エラー）が起きないよう、描画関数 `drawTextWithStroke` で安全に処理を例外ハンドリングしています。
